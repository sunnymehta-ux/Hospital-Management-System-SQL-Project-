# Hospital-Management-System-SQL-Project-

##📌 Overview

This project is a Hospital Management System built using MySQL, designed to manage and analyze hospital operations such as patient records, doctor schedules, appointments, treatments, and billing.  The goal of this project is to simulate real-world healthcare data handling and demonstrate strong SQL skills.
---

## 🎯 Objectives

* Design a normalized relational database
* Manage patient and doctor data efficiently
* Track appointments and treatments
* Handle billing and payments
* Generate insights using advanced SQL queries

---

## 🗂️ Database Schema

The system consists of the following tables:

* **patients** → Stores patient details and insurance info
* **doctors** → Stores doctor details and specialization
* **appointments** → Links patients with doctors
* **treatments** → Records treatments per appointment
* **billing** → Handles payment and financial data

---

## 🔗 Relationships

* One patient can have multiple appointments
* One doctor can attend multiple patients
* Each appointment can have multiple treatments
* Billing is linked to both patients and appointments

---

## ⚙️ Technologies Used

* MySQL(JOIN, WHERE, HAVING, Aggregations functions, GROUP BY, ORDER BY)
* MySQL Workbench

---

## 📊 Key Features / SQL Queries

### 🔥 1. Busiest Doctors

Identifies doctors with the highest number of appointments.

### 💰 2. Revenue Analysis

Calculates total revenue generated by each doctor.

### ⏱️ 3. Patient Visit Frequency

Finds patients with frequent hospital visits (chronic cases).

### 📅 4. Daily Revenue Trends

Tracks hospital income over time.

### 🧠 5. Full Patient History

Retrieves complete medical and billing history of a patient.

---



## 🚀 How to Run

1. Open MySQL Workbench
2. Run schema.sql to create database and tables
3. Import your dataset (CSV files)
4. Execute SQL queries
